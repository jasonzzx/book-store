# Anatomy of a Book Folder

A book folder looks like this:

```
Books/
  my-book/
    book.json
    chapters/
      01.md
      02.md
      03.md
```

## The manifest

`book.json` is the book's identity card:

```
{
  "id": "my-book",
  "title": "My Book",
  "author": "Your Name",
  "description": "One or two sentences shown on the shelf and the book page.",
  "language": "en",
  "chapters": [
    { "id": "01", "title": "First Chapter", "file": "chapters/01.md" },
    { "id": "02", "title": "Second Chapter", "file": "chapters/02.md" }
  ]
}
```

A few rules keep everything predictable:

- The `id` must match the folder name, and should be lowercase with hyphens — it becomes the book's URL.
- Chapters appear in the order they are listed in the manifest, not alphabetical order. Reordering chapters is a one-line edit.
- The `file` path is relative to the book folder.

## The chapters

Chapters are plain Markdown. The reader supports the parts of Markdown that matter for prose:

- Headings (`#`, `##`, `###`) — the reader uses them for structure within a chapter.
- Paragraphs, separated by blank lines.
- *Italics* and **bold**.
- Blockquotes (`>`) — nice for epigraphs and morals.
- Lists, like this one.
- Code blocks, for technical books.
- Horizontal rules (`---`) as scene breaks.

## The cover

You may have noticed there are no image files anywhere. Covers are generated automatically: the app hashes each book's title into a color palette and typesets the title and author over it. Every book gets a distinctive, consistent cover with zero artwork — and the same book always gets the same cover, on every device.
