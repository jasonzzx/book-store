# How This Store Works

Welcome. The app you are reading this in has no database, no CMS, and no admin panel. Its entire library is a folder called `Books` in a git repository.

Every book in the store is a subfolder. Each subfolder contains a small `book.json` file describing the book — its title, author, description, and chapter list — plus one Markdown file per chapter. When the site is built and deployed, the app scans the `Books` folder, reads every manifest, and generates the shelf you saw on the home screen.

That is the whole architecture. There is nothing else to it.

## What that buys you

Because the library is just files in a repository:

- Adding a book is a git commit. Push a new folder, and the next deployment puts it on the shelf.
- Every book is version-controlled. Typo fixes, new chapters, and rewrites all have history, diffs, and blame — the same tools you would use for code.
- The content is portable. Markdown files are readable in any editor, on any platform, forever. There is no export problem because there was never an import.
- Anyone can contribute through the normal review workflow. A new book can arrive as a pull request.

## What stays on your device

The store deliberately keeps nothing about you on any server. Your reading progress, your bookmarks, and your reader settings — font size and theme — are stored in your browser's local storage, on your device only.

This has one honest consequence worth knowing: your progress does not follow you between devices or browsers, and clearing your browser data clears it. That is the trade against needing no account and no database at all.
